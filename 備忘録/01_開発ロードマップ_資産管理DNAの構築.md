# 備忘録 01: 開発ロードマップ - 資産管理DNAの構築

## 1. 目的とビジョン
**「慎重に、かつ機械的に、個人の感情を排した資産管理システムの構築」**
古文プロジェクトで培った「フルオート開発環境」を基盤に、実際の資産（株式・仮想通貨等）を管理・分析する堅牢なツールを作り上げる。

## 2. 開発フェーズ (Milestones)
### フェーズ 1: スステルス基盤の構築
- [ ] Commanderシステムの完全動作確認
- [ ] Gitリポジトリの初期化と自動同期（GitHub/Private）
- [ ] 資産管理DNAの深掘り（リスク許容度、データ保護ルールの策定）

### フェーズ 2: データ収集モジュールの開発
- [ ] 金融データAPI（Yahoo Finance等）のスクレイピング/API連携
- [ ] CSV/Excel等からの過去データインポート機能
- [ ] データの同一性検証プロトコルの実装

### フェーズ 3: 分析エンジンとUIの実装
- [ ] バックテストエンジンの構築
- [ ] Streamlitによる「ダッシュボード」の実装
- [ ] ポートフォリオの可視化と再配分（リバランス）提案機能

## 3. StockProject 専用の「機械的ルール」
- **データ不変の原則**: 取得した生データ（Raw Data）は決して直接加工せず、必ずコピー（加工用データ）を用いて処理する。
- **承認の壁抜け**: すべての OS コマンドは `_order_*.json` を経由し、VS Code のポップアップを 100% 回避する。
- **スナップショットの徹底**: 重要なデータベース更新前には、必ずファイルベースのバックアップを作成する。
