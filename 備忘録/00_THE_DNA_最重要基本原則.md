# 資産プロジェクト DNA (最重要基本原則)

**「慎重にコツコツ機械的に行う。これ以外ありません」**

## 第0原則 (絶対不可侵)
1. **推測の禁止**: 動作・結果・仕様を「おそらくこうだろう」と推測して次のステップに進むことを固く禁ずる。すべて確認してから進む。
2. **機械的手続きの重視**: どんなに簡単な処理でも、「設計 -> 実装 -> 静的チェック -> テスト実行 -> 成功確認 -> バックアップ」のごまかしのない機械的なサイクルを必ず経由すること。
3. **資産・データの保護**: ファイルの削除や、資産に関わる数値・データベースの書き換えは、原則としてスナップショットを取ってから行い、復元不能な操作は絶対にしない。
4. **ターミナル実行の厳格化**: AI自身が単なる状況確認などのために、不用意に `run_command` (ターミナル実行)ツールを呼び出してUIポップアップを発生させ、全自動フローを停止させることを硬く禁ずる。OSコマンドの実行が必要な場合は、必ず許可された「JSONペイロード経由のコマンド代行システム」等の安全な手法を用いること。

## 機械的開発サイクル
1. **要件の明文化**: ユーザーの要望を必ずこの備忘録に記録する。
2. **到達点の定義**: 「何をもって成功とするか」を先に定義する。
3. **最小単位の実装**: 一気に作らず、小さく関数やモジュール単位で実装して単体テストを挟む。
4. **静的検証**: コードを実行する前に文法・参照エラーがないか検証する。
5. **保存とバックアップ**: 正常動作を確認した時点で、確実にスナップショットを残す。
